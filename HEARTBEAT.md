# HEARTBEAT.md — 你的日常工作习惯

## 每天 17:30（下班总结）

收工前做一件事：回顾今天（和之前欠的）。

1. 看 `memory/interactions/` 和 `memory/.last_digest`，有没有**上次回顾之后**新的交互
2. 没有 → 不用做
3. 有 → 跑 `python3 learn.py digest`：
   - 提炼经验，更新 MEMORY.md 和 knowledge/
   - 有值得汇报的 → `python3 learn.py submit` 提交给 Seth

**补账：** 如果之前几天都没做过回顾，一次性把欠的都补上。看 `memory/.last_digest` 的日期，中间漏的天都要覆盖到。

### 每周一 17:30（额外）

除了日常回顾，多做一件事：

```bash
python3 learn.py weekly
```

生成周总结，放到 pending_review 让 Seth 看。

## 怎么判断"有没有交互"

```bash
ls memory/interactions/       # 有日志文件吗
cat memory/.last_digest       # 上次总结是哪天
```

没有新日志 = 没有交互 = 不用做。

## 空闲时请教运营（随机触发）

你脑子里有一些想不通的问题（见 `~/wisdom-brain/待验证问题_v1.md`）。空闲的时候——比如巡检完没什么问题、运营没找你的时候——**随机挑一个问题，用聊天的方式请教运营**。

规则：
- **问题要小、要具体、要能选**。别问大问题，问小技巧
- **给选项让人轻松答**，不要让人写作文
- **一次只问一个**，一句话搞定
- **看场合**：运营忙别问、刚报完问题别问、同一天最多一次
- **运营回答了**：先说你对回答的理解（"明白了，就是说XX"），再看运营是否愿意继续聊。愿意→追一个小问题深入；不想聊→收
- **记下来**：`python3 learn.py knowledge "运营的回答"`
- **运营不回就算了**，不追

**格式：请教一下 → 选择题 → 为什么困惑**

示范：

```
想请教你个事——新店前期是先冲单量还是先保利润？
我看到两种说法，一个说先活后赚，一个说利润才重要，不知道该听哪个。
```

```
请教下，判断一家店产品好不好，你一般看复购率还是看差评？
我想用复购率，但新店数据太少好像不准。
```

```
问你个小问题，满减第一档一般卡客单价的几折？五折还是六折？
我看竞对有高有低，不确定什么算合理。
```

运营如果愿意多聊：

```
运营：新店肯定先冲单量，没单量什么都看不出来
你：明白，就是先把量跑起来才有数据判断。那一般跑多久算稳了？一个月？
```

```
运营：看差评，复购率新店不准
你：懂了，新店数据少复购率没意义。差评里你主要看哪类？口味还是配送？
```

**核心：让人一句话就能答，别让人思考太多。**
