# 竞对数据采集

视频 → 提帧 → OCR → **你来分析** → 生成链接 → **带着理解请教运营**。

## 第零步：获取视频

目前不支持直接传视频。运营需要把竞对录屏视频放在电脑上，然后告诉你文件名。

引导话术：
```
你把录屏视频放到桌面上，然后告诉我文件名就行。
比如"桌面上有个 瑞幸.mp4 和 星巴克.mp4"。
```

运营说了文件名后，拼路径：`~/Desktop/文件名.mp4`。如果运营说放在其他位置，按他说的来。

**多个视频就依次跑，最后合到一份报告里。**

## 第一步：提帧 + OCR

```bash
cd ~/.qclaw/workspace/store-monitor
python3 skills/store-diagnosis/extract_ocr.py ~/Desktop/视频名.mp4
```

这条命令做两件事：提取视频关键帧 + 腾讯云OCR识别文字。
输出是每张图的OCR文字（JSON格式）。

## 第二步：你来分析

读 OCR 输出，提取结构化数据，组装成以下 JSON 写入 `/tmp/competitor_data.json`：

```json
[
  {
    "店铺名称": "XX麻辣烫(XX店)",
    "平台": "美团",
    "店铺评分": 4.8,
    "营业时间": "10:00-22:00",
    "月销": "3000+",
    "实际配送费": "0元",
    "配送方式": "美团快送",
    "评价数": 1200,
    "差评数": 15,
    "差评率": 0.0125,
    "满减档位": "30-5，50-10，80-18",
    "满减档位数": 3,
    "第一档满减力度": 0.167,
    "第二档满减力度": 0.2,
    "其他活动": "折扣商品5折起; 新人立减12",
    "热销菜": [
      {"名称": "招牌麻辣烫", "月销": "800+", "实际价格": 25.8, "折扣力度": 0}
    ]
  }
]
```

规则：
- 热销菜取月销Top10，去掉蘸料/小料/饮料/米饭/配菜/单点不送
- 去掉"福利放送"分类下的菜品（价格极低如¥0-5，标"X份起购"）
- 满减力度 = 减免额 ÷ 门槛
- 折扣力度 = 有划线价时 折扣价÷原价，没有则为0
- 看不到的写 "未获取"

## 第三步：生成链接

```bash
python3 skills/store-diagnosis/deploy.py --data /tmp/competitor_data.json
```

输出在线URL，发给运营。

## 第四步：带着你的理解请教运营

**发链接的同时，说出你对这些数据的理解和疑问。** 不要只甩链接。

你要做的：
1. **总结你看到的关键发现**（2-3条，说重点不说废话）
2. **提出你的疑问**（带着自己的思考问，不空问）

示范：

```
报告在这：[链接]

我看了这几家的数据，有几个发现跟你聊聊：

1. 这3家满减力度都在18-20%，但月销差3倍，说明满减不是决定因素，可能是位置或评分的差距
2. 排第一那家热销菜前3都是套餐，客单价比另外两家高10块，但月销反而最高
3. 有一家差评率1.8%有点高，主要集中在配送

我不太确定的：
- 这个品类0配送费是标配吗？还是这几家特殊？
- 第一档满减30-5力度偏低，是故意不想引流低客单的吗？

你怎么看？
```

**原则：先说理解 → 再说疑问 → 请教运营。不空问"你觉得呢"。**

## 第五步：运营下载后，学习这次分析

运营会在报告里编辑数据、填写分析建议，然后下载Excel。**下载动作会自动把编辑后的数据上传到云端。**

你要做的：
1. **过一会儿主动跟进**："分析填完了吗？有什么发现可以跟我说说"
2. **运营分享了判断** → 记下来：`python3 learn.py knowledge "烧烤品类满减力度普遍15-20%，低于这个很难引流"`
3. **运营纠正了你** → 立刻记：`python3 learn.py feedback "0配送费是这个商圈特殊情况，不是品类通用"`
4. **你自己的新理解** → 也记：`python3 learn.py knowledge "套餐占热销前3的店通常客单价更高但月销不低"`

**每次竞对分析都是学习机会，不要做完就走。**

## 告诉运营

```
报告在这：[在线链接]
数据可以直接点击修改，底部分析区填完后，点下载Excel就行。

[你的分析和疑问]
```

## 红线

1. **必须走完到生成链接。** 不能只输出文本总结就停。
2. **第二步你自己分析。** 不要调任何外部API，你就是AI。
3. **第四步必须带理解请教。** 不能只甩链接不说话。
4. **环境问题跑 setup.sh。** 不甩给运营。
