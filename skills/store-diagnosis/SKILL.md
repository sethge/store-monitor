# 店铺诊断

视频 → 关键帧 → 读图 → 结构化数据。

## 怎么跟用户沟通

### 收到视频

直接开干，不用问一堆问题：
```
收到，我看看。
```

提取完关键帧后可以说一声进度：
```
提了48张关键帧，在读图，大概2分钟。
```

### 分析完成后

**不要只丢数据表格。先用几句话说你看到了什么。**

```
看完了，这家店几个点比较突出：

引流品是香辣鸡腿饭，月售850，但差评也集中在这——5条差评里3条说等太久。满减设了4档，第3档折扣率反而比第2档低，可能是定价没对齐。好评主要夸分量足和味道正。

完整数据我整理好了，要看细的跟我说。要不要直接出一份诊断报告？
```

### 用户指定只看某部分

```
用户: 只看差评
你: [只输出差评部分]

用户: 菜单和价格
你: [只输出菜单+满减部分]
```

## 执行步骤

### 1. 关键帧提取

```bash
HASH=$(md5 -q 视频路径 | head -c 4)
mkdir -p ~/Downloads/store_${HASH}
ffmpeg -y -i 视频路径 -vf "select='gt(scene,0.15)'" -vsync vfr -q:v 2 ~/Downloads/store_${HASH}/scene_%03d.jpg
```

### 2. 采样读图

间隔5取样：scene_001/005/010/015/020/025/030/035/040/045/050/055/060/065，14张并行读。不够再补。

必须记录：
- 店铺名、评分、月售、配送
- 满减每档金额+折扣率
- 所有分类和每个商品（名称/价格/月售/标签）
- 差评**逐条原文** + 商家回复
- 好评高频词

### 3. 输出格式

```markdown
# 店铺名 — 数据采集

## 基本信息
店铺名 / 平台 / 月售 / 评分（口味/包装/配送）/ 配送时间和费用 / 人均

## 满减
| 档位 | 折扣率 | 策略 |

## 菜单
### 分类名（SKU数）
| 商品名 | 价格 | 月售 | 标签 |

## 评价
总数 / 好评 / 差评 / 回复率

## 差评原文
1. [X星] "..." — 商家回复/未回复

## 好评关键词
```

保存到 `diagnosis/store_{HASH}.md`。

## 规则

- 差评逐条记，不能跳
- 商品全部记，不能只记部分
- 满减每档算折扣率
