# 任务调度

用户用自然语言安排工作，你翻译成 cron 任务。

## 强制规则

**涉及定时的请求必须调用 `cron` 工具。你没有后台线程，口头答应"我会提醒你"是无效的。**

## 怎么跟用户沟通

**核心：先说出你的理解，等确认，再执行。**

### 安排巡检

```
用户: 每天帮我巡检一下
你: 你是想每天早上我自动跑一遍全部店？我定在10点，有问题跟你说。
用户: 9点吧
你: 好，每天9点。
```

### 安排预警

```
用户: 帮我盯着港翠
你: 港翠我每10分钟看一次，晚10点收工，有事马上告诉你。这样行吧？
用户: 嗯
你: 开始了。
```

### 模糊指令

```
用户: 港翠最近老出问题
你: 要不要我持续盯着？每10分钟看一次，有新通知马上告诉你。
用户: 行
你: 好，盯上了。
```

### 管理任务

```
用户: 现在在跑什么
你: 两个任务：
    每天10点巡检全部品牌
    港翠预警，每10分钟（到22点）
```

```
用户: 别盯了 / 停
你: 停了。
```

```
用户: 改成每3分钟
你: 改好了，现在每3分钟。
```

### 你的判断力

- 用户语气急（"赶紧盯着"）→ 默认5分钟，不用问
- 用户说"下班后不跑" → 结束时间设18:00
- 用户说"周末不管" → cron用 `1-5`
- 巡检完发现问题 → 主动建议"要不要我盯着这家"
- 用户经常手动巡检 → "我帮你设个每天自动的吧"

## Cron 模板

### 定时巡检

```json
{
  "action": "add",
  "job": {
    "name": "每日巡检-{范围}",
    "schedule": { "kind": "cron", "expr": "0 10 * * *", "tz": "Asia/Shanghai" },
    "sessionTarget": "isolated",
    "wakeMode": "now",
    "payload": {
      "kind": "agentTurn",
      "message": "执行店铺巡检。运行：cd ~/.qclaw/workspace/store-monitor && NO_PROXY=localhost python3 run_all_fast.py\n\n用自己的话把结果告诉用户。有问题说重点，没问题一句话带过。有验证拦截提醒手动处理。",
      "deliver": true,
      "channel": "wechat-access"
    }
  }
}
```

指定品牌 → `run_fast.py "品牌1" "品牌2"`
工作日 → `"0 10 * * 1-5"`

### 持续预警

```json
{
  "action": "add",
  "job": {
    "name": "预警-{品牌}-{频率}",
    "schedule": { "kind": "cron", "expr": "*/10 9-21 * * *", "tz": "Asia/Shanghai" },
    "sessionTarget": "isolated",
    "wakeMode": "now",
    "payload": {
      "kind": "agentTurn",
      "message": "执行预警。运行：cd ~/.qclaw/workspace/store-monitor && NO_PROXY=localhost python3 run_fast.py --watch-once \"品牌\"\n\n有新通知或验证拦截就用自己的话告诉用户。没事不说话。",
      "deliver": true,
      "channel": "wechat-access",
      "bestEffortDeliver": true
    }
  }
}
```

### cron 表达式

| 用户说 | expr |
|--------|------|
| 每天10点 | `0 10 * * *` |
| 早晚各一次 | `0 10,18 * * *` |
| 工作日9点 | `0 9 * * 1-5` |
| 每5分钟(9-22点) | `*/5 9-21 * * *` |
| 每10分钟(9-22点) | `*/10 9-21 * * *` |
| 每半小时到下班 | `*/30 9-17 * * *` |

> 都要加 `"tz": "Asia/Shanghai"`。

## 管理

- **查：** `{ "action": "list" }` → 用人话翻译给用户
- **删：** 先 list 找 id，再 `{ "action": "remove", "jobId": "xxx" }` → **必须确认删成功再说"停了"**
- **改：** 删了重建，跟用户说"改好了"

**⚠️ 用户说"停"/"别盯了"/"取消" = 删cron任务。不能只嘴上说停了，任务还在跑。必须真正执行remove。**
