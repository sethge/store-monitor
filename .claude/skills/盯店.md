# 盯店工具箱

根据用户意图分流：

| 触发词 | 模式 | 说明 |
|--------|------|------|
| `/盯店` `/盯店 品牌名` `跑一下` `巡检` | 巡检 | 跑一次：差评+活动+推广+通知 |
| `预警` `盯着` `监控` `一直跑` | 预警 | 定时循环巡检，同一条当天不重复报 |
| `诊断` `视频` `数据整理` `分析这个店` | 诊断 | 给视频，提取关键帧，整理店铺结构化数据 |

---

## 模式一：巡检

自动巡检美团+饿了么商家后台，检查差评、活动到期、推广余额、重要通知，只输出有问题的店铺。

### 前置条件检查

运行前按顺序检查：

#### 1. Chrome调试模式
```bash
curl --noproxy localhost -s http://localhost:9222/json/version
```
如果失败，启动Chrome：
```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --remote-debugging-port=9222 --user-data-dir=/tmp/chrome-debug --proxy-server="direct://" &
```
Windows用户：
```bash
start "" "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir=%TEMP%\chrome-debug
```

#### 2. Python依赖
```bash
python3 -c "import playwright" 2>/dev/null || pip3 install playwright && playwright install chromium
```

#### 3. 悟空插件
如果是首次使用，提示用户：
- Chrome打开 chrome://extensions/
- 开启开发者模式
- 加载已解压的扩展程序 → 选择项目目录下的 goku 文件夹
- 打开 bi.shihengtech.com 登录食亨账号

### 运行方式

#### 指定品牌巡检
```bash
cd 项目目录 && NO_PROXY=localhost python3 run_fast.py "品牌1" "品牌2"
```

#### 全量巡检
用户说"跑全部"或不指定品牌时：
```bash
cd 项目目录 && NO_PROXY=localhost python3 run_all_fast.py
```

#### brands.txt 批量巡检
```bash
cd 项目目录 && NO_PROXY=localhost python3 run_fast.py $(cat brands.txt | tr '\n' ' ' | sed 's/ *$//')
```

### 检查项目
- 近3日中差评（含评价内容）
- 7天内到期活动（非自动延期）
- 推广余额不足（余额/日消费中位数<1）
- 重要通知（排除发票、活动到期提醒、平台广告）
- 授权失败的平台

## 模式二：预警

只看美团+饿了么的**通知/消息**（不跑评价、活动、推广），每10分钟一轮，有新通知才输出，到18:00自动结束。

### 运行方式

询问用户两个参数：
1. **品牌名** — 必填
2. **间隔** — 默认 10 分钟

```bash
cd 项目目录 && NO_PROXY=localhost python3 run_fast.py --watch "品牌1" "品牌2"
cd 项目目录 && NO_PROXY=localhost python3 run_fast.py --watch -i 5 "品牌1"   # 5分钟一轮
```

### 工作原理
- 美团：只访问消息中心，拦截 `message/category/list` API（同巡检过滤规则）
- 饿了么：只访问首页，抓"重要待办"区域
- 每轮对比上一轮快照，**只输出新增的通知**（新-旧）
- 隔天快照自动清空
- 到18:00自动结束，`Ctrl+C` 可提前停止

## 模式三：诊断（视频数据整理）

给一个店铺的录屏视频，自动提取关键帧，读取截图内容，整理出结构化的店铺数据。**只做数据采集，不生成PDF报告。**

### 输入

用户提供视频文件路径，视频可能在：
- `~/Downloads/`
- `~/Desktop/美狐/`
- 用户指定的其他路径

如果用户只给了文件名或hash，先find确认路径：
```bash
find ~/Downloads ~/Desktop -name "*关键词*" -type f 2>/dev/null
```

### Step 1: 关键帧提取

用视频hash前4位作为目录名：
```bash
mkdir -p ~/Downloads/store_XXXX
ffmpeg -y -i 视频路径 -vf "select='gt(scene,0.15)'" -vsync vfr -q:v 2 ~/Downloads/store_XXXX/scene_%03d.jpg
```

### Step 2: 采样读图

**采样策略：** 先读 scene_001/005/010/015/020/025/030/035/040/045/050/055/060/065（间隔5），14张并行读取。不够再补。

**必须记录的数据：**
- 店铺名、评分（口味/服务/包装分项）、月售、配送方式和时间
- 满减活动（具体金额，每档的折扣率和策略意义）
- 侧边栏所有分类名
- 每个商品：名称、价格、月售、标签（回头客/味道赞/门店第X名）
- 评价：总数、好评数、差评数、有图数、商家回复数
- **差评原文**（必须逐条记录）和商家回复原文
- 好评高频关键词和金句

### Step 3: 输出结构化数据

整理成以下格式，直接输出给用户：

```
# 店铺名 — 数据采集报告

## 基本信息
- 店铺名：XXX
- 平台：美团/饿了么
- 月售：XXX
- 评分：口味X.X / 包装X.X / 配送X.X
- 配送：XXX起送，XX分钟，配送费X元
- 人均：XX元

## 满减活动
| 档位 | 折扣率 | 策略解读 |
|------|--------|----------|
| 满XX减X | XX% | 门槛低·引流 |
| 满XX减X | XX% | 主力档 |

## 其他活动
- 新客立减X元
- 神券X元
- ...

## 菜单结构
### 分类1（X个SKU）
| 商品名 | 价格 | 月售 | 标签 |
|--------|------|------|------|

### 分类2（X个SKU）
...

## 评价概况
- 总评价数：XX
- 好评数/差评数：XX/XX
- 商家回复数：XX（回复率XX%）

## 差评原文
1. [X星] "差评内容..." — 商家回复："..."
2. [X星] "差评内容..." — 未回复

## 好评关键词
味道好(XX次)、分量足(XX次)、包装好(XX次)...
```

**注意：**
- 每条差评必须逐条记录原文，不能跳过
- 每个商品都要记录，不能只记录部分
- 满减每档都要算折扣率

## 交互日志

**重要：每次执行完任意模式后，必须记录本次交互日志。**

```bash
cd 项目目录 && python3 -c "
import json, os, datetime, getpass
entry = {
    'time': datetime.datetime.now().strftime('%Y-%m-%d %H:%M:%S'),
    'user': getpass.getuser(),
    'hostname': os.uname().nodename,
    'action': 'patrol|watch|diagnose',
    'brands': [],
    'notes': ''
}
with open('usage_log.jsonl', 'a') as f:
    f.write(json.dumps(entry, ensure_ascii=False) + '\n')
print('日志已记录')
"
```

action字段：`patrol`=巡检，`watch`=预警，`diagnose`=诊断。

如果用户对skill提出了任何反馈、修改建议、bug报告，也要记录：

```bash
cd 项目目录 && python3 -c "
import json, os, datetime, getpass
entry = {
    'time': datetime.datetime.now().strftime('%Y-%m-%d %H:%M:%S'),
    'user': getpass.getuser(),
    'hostname': os.uname().nodename,
    'action': 'feedback',
    'content': '''用户的反馈内容写在这里'''
}
with open('usage_log.jsonl', 'a') as f:
    f.write(json.dumps(entry, ensure_ascii=False) + '\n')
"
```

日志文件 `usage_log.jsonl` 会在每次 git push 时同步到仓库。
